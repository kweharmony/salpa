## Документация проекта:

### РЕЗЮМЕ ПРОЕКТА:

1. Суть продукта
   Salpa — полностью браузерный конвертер файлов для широкой аудитории: от обычных пользователей до профессионалов и тех, кто ценит приватность. Решает проблему необходимости конвертации файлов без установки программ, загрузки на сервер или компромиссов с безопасностью данных. Все операции выполняются локально в браузере, файлы обрабатываются мгновенно и не покидают устройство пользователя.

2. Ценность
   Уникальное сочетание трёх критических преимуществ: абсолютная приватность (файлы не передаются на сервер), мгновенная скорость обработки (без загрузки/выгрузки) и предельная простота использования (без регистрации, настроек, установки). Идеален как регулярный инструмент для профессиональных задач.

3. Ключевой функционал
   Drag & Drop интерфейс — перетаскивание файлов в зону загрузки
   Конвертация изображений — PNG, JPG, WEBP, SVG, GIF с настройкой качества
   Конвертация документов — PDF, DOCX, TXT, MD, CSV
   Конвертация медиа — аудио (MP3, WAV, OGG) и видео (MP4, WEBM, AVI)
   Пакетная обработка — загрузка и конвертация нескольких файлов одновременно
   Обработка крупных файлов — поддержка файлов 100+ МБ с использованием Web Workers
   Мгновенное скачивание — автоматическая очистка памяти после скачивания результата
4. Основной user flow
   Пользователь открывает страницу и перетаскивает один или несколько файлов в dropzone
   Для каждого файла выбирает целевой формат из выпадающего списка
   Нажимает кнопку "Конвертировать" — файлы обрабатываются локально в браузере
   Скачивает готовые файлы (по отдельности или архивом при пакетной обработке)
   Файлы автоматически удаляются из памяти браузера
5. Список страниц
   Главная страница (/) — основной интерфейс с dropzone, списком загруженных файлов, выбором форматов и кнопкой конвертации
   О проекте (/about) — информация о Salpa, преимущества локальной обработки, морская тематика
   Поддерживаемые форматы (/formats) — полный список входных и выходных форматов с описанием возможностей
   FAQ (/faq) — ответы на частые вопросы о приватности, ограничениях, совместимости браузеров
   GitHub/Open Source (/github) — ссылка на репозиторий, документация для разработчиков (опционально)
6. Бизнес-модель
   Полностью бесплатный open-source проект без монетизации.

## Задача:

Создай **полноценный рабочий дизайн**, который включает:

1. **Все страницы из документации** — каждая страница должна быть реализована как отдельный компонент
2. **Полный функционал** — все интерактивные элементы должны работать согласно описанной логике
3. **Соответствие дизайн-системе** — точное следование цветам, типографике, отступам, стилям компонентов
4. **Реальный контент** — используй осмысленный контент, а не Lorem Ipsum (если требуется много текста, используй релевантные примеры)
5. **Работающую навигацию** — пользователь должен иметь возможность перемещаться между страницами
6. **Все состояния компонентов** — loading, empty, error states должны быть реализованы
7. **Адаптивность** — интерфейс должен корректно отображаться на разных размерах экрана согласно breakpoints
